# MeshCore Operations

These operator checklists focus on fixed MeshCore infrastructure for New
England: Repeaters, Room Servers, and optional MQTT observer setups.

## Understand MeshCore roles

MeshCore **Companion** devices are personal or local nodes. They are commonly
carried, used at home with a computer, or installed in a car. Most communicate
with the MeshCore app over Bluetooth or USB; some use Wi-Fi.

MeshCore **Repeaters** and **Room Servers** are the infrastructure devices to
plan carefully. They should be stable, reachable, and documented because other
operators may depend on them for coverage and routing.

## Set up MeshCore Repeaters and Room Servers

1. Pick a legal site with good elevation, weather protection, stable power, and
   safe antenna mounting.
2. Use supported MeshCore hardware and record the device, antenna, power source,
   maintainer, and install location.
3. Flash the current stable MeshCore firmware and record the version.
4. Configure the correct MeshCore role for the site: Repeater or Room Server.
5. Test locally before installing at height.
6. After deployment, verify coverage from multiple directions and watch for
   channel congestion.

Useful links:

- [MeshCore docs](https://docs.meshcore.io/)
- [MeshCore flasher](https://flasher.meshcore.io/)
- [MeshCore map](https://map.meshcore.io/)

## Publish observer data with mctomqtt

MeshCore observer setups publish what a local node can hear to MQTT so maps and
analysis tools can show regional activity. A typical observer is a MeshCore node
connected to a small always-on computer running `mctomqtt`.

You do not need to pair every Repeater or Room Server with an observer. New
England Mesh already has many observers on the main mesh, so add another one
when it improves regional visibility or fills a known monitoring gap.

Basic process:

1. Build or install `mctomqtt` on a small always-on host near the radio.
2. Connect the host to the MeshCore node over the supported serial or network
   interface.
3. Configure the MQTT broker, topic prefix, observer identity, and credentials.
4. Start it under a service manager such as `systemd`.
5. Confirm that observations appear in the intended map or downstream service.
6. Document the radio, host, antenna, power source, and maintainer.

Links to expand from:

- [mctomqtt on GitHub](https://github.com/liamcottle/mctomqtt)
- [New England MeshCore map](https://map.newenglandme.sh/)
- [MeshCore region boundaries](/regions)

## Community MQTT services

New England Mesh operates a combined MQTT backend for Connecticut,
Massachusetts, and New Hampshire data used by the [regional
map](https://map.newenglandme.sh/) and
[analyzer](https://analyzer.newenglandme.sh/). That backend does **not** accept
direct observer uploads. Operators should publish to their local community's
MQTT service; the regional services aggregate that community data upstream.

MQTT infrastructure is operated by the individual regional communities, not by
New England Mesh. Follow the community's current instructions and access policy
before connecting an observer:

- **Greater Boston Mesh:** [`mqttmc01.bostonme.sh` setup and broker
  details](https://bostonme.sh/docs/MeshCore/meshcore-mqtt). The documented
  listener uses secure WebSockets on port `443` with token authentication.
- **Connecticut Mesh:** [CT Mesh MeshCore
  resources](https://ctmesh.org/meshcore-resources). CT Mesh documents an
  uplink-only broker at `mqtt.ctmesh.org` for fixed observer nodes, using the
  `BDL` region code.
- **NHMesh:** [NHMesh knowledge base](https://nhmesh.live/kb). Use its current
  Observer / Gateway Setup and MQTT guidance for New Hampshire deployments.
- **Maine Mesh:** [Maine Mesh observer
  configuration](https://github.com/deejdeej/MaineMesh-Meshcore/blob/main/observer/README.md).
  Maine's documented `BGR` observer configuration publishes to the Greater
  Boston Mesh broker in addition to its other monitoring destinations.

Broker credentials and connection requirements can change. Use the linked
community documentation rather than copying credentials into this site.
