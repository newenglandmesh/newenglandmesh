# Deployment

The site is configured for Cloudflare Pages.

## Branch mapping

| Branch | Cloudflare Pages deployment | Custom domain |
| --- | --- | --- |
| `main` | Production | `https://newenglandme.sh/` |
| `staging` | Preview/staging | `https://staging.newenglandme.sh/` |

## Cloudflare Pages settings

Create a Cloudflare Pages project connected to this GitHub repository and use
these settings:

- Framework preset: Docusaurus, or None with the values below
- Production branch: `main`
- Preview branch include list: `staging`
- Build command: `npm run build`
- Build output directory: `build`
- Root directory: `/`
- Node.js version: `20` or newer

## Custom domains

Add both domains in the Cloudflare Pages project:

- `newenglandme.sh` for the production deployment
- `staging.newenglandme.sh` for the staging branch deployment

Because the DNS zone is already in Cloudflare, Cloudflare Pages can create or
verify the required DNS records from the Pages dashboard.

## URL selection

Cloudflare Pages exposes `CF_PAGES_BRANCH` during builds. The Docusaurus config
uses that value to set the canonical site URL:

- `staging` branch builds use `https://staging.newenglandme.sh`
- all other builds default to `https://newenglandme.sh`

For local verification, use:

```bash
npm run build:production
npm run build:staging
```

You can override the URL explicitly if needed:

```bash
SITE_URL=https://example.com npm run build
```
