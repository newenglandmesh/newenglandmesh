# Regional communities and services

New England Mesh provides cross-state coordination, documentation, maps, and
analysis tools. Local communities operate their own Discord servers, setup
guides, maps, MQTT services, and infrastructure. Use the community closest to
you for local radio settings, MQTT uploads, and site-specific questions.

## New England Mesh regional services

These services are hosted for the wider New England community:

- [New England Mesh website and documentation](https://newenglandme.sh/)
- [New England MeshCore live map](https://map.newenglandme.sh/)
- [New England MeshCore analyzer](https://analyzer.newenglandme.sh/)
- [MeshCore region boundaries](https://newenglandme.sh/regions/)
- [Service healthcheck](https://healthcheck.newenglandme.sh/)
- [New England Mesh Discord](https://discord.newenglandme.sh/)
- [New England Mesh GitHub repository](https://github.com/newenglandmesh/newenglandmesh)

The regional map and analyzer combine data for Connecticut, Massachusetts, and
New Hampshire. Their backend does **not** accept direct observer uploads.
MeshCore observers should publish to their local community MQTT service; New
England Mesh aggregates that community data upstream.

## Choosing an MQTT service

Publish observer or node data to the MQTT service operated by your local
community. Local maintainers define credentials, topic conventions, region
codes, and uplink policies. See [MeshCore Operations](./meshcore-operations) and
[Meshtastic Operations](./meshtastic-operations) for protocol-specific guidance.

If a community link or service changes, please update this page through the
[documentation contribution guide](./contributing).
